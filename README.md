# BrowserStack-Ecommerce-Manual-Testing
# E-commerce Manual Testing Project

## Overview
This is a complete manual testing project for an e-commerce application created using industry standard testing practices.

The project focuses on requirement-based testing, scenario design, detailed test cases and traceability.

## Reference
Test scenarios, coverage and test areas are derived from  
**BrowserStack – E-commerce App Testing – Techniques and Approaches guide.**

## Scope
The scope of this project includes:
- Functional coverage of major e-commerce modules
- Negative and edge case validation
- Requirement to test case traceability

## Artifacts Included

01_FunctionalModules.xlsx  
02_TestScenarios.xlsx  
03_TestCases.xlsx  
04_NegativeEdgeTestCases.xlsx  
05_RTM.xlsx  

## Artifact Description

- 01_FunctionalModules.xlsx  
  Contains functional modules and sub-modules of the e-commerce system.

- 02_TestScenarios.xlsx  
  Contains high level manual test scenarios mapped to modules.

- 03_TestCases.xlsx  
  Contains detailed manual test cases with preconditions, steps, test data and expected results.

- 04_NegativeEdgeTestCases.xlsx  
  Contains negative and edge test cases derived from scenarios.

- 05_RTM.xlsx  
  Requirement Traceability Matrix mapping requirements to scenarios and test cases.

## Testing Approach
- Requirement based testing
- Scenario based test design
- Risk and edge case identification
- Traceability driven coverage

## Tools Used
- Microsoft Excel
- Documentation based analysis

## Key Highlights
- Clear module to scenario mapping
- Separate negative and edge case coverage
- Complete RTM for test coverage validation
- Industry reference based test design
